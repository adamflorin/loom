Loom is a collection of Max For Live devices which generate MIDI.

* Live 9 + Max For Live required

To install, open Loom.alp and install it wherever you like.

I like to put it next to the User Library and then Add Folder... in Live's Browser to point to it.

The Loom pack contains players and modules.

A player is a group of one or more modules in a MIDI Effect Rack.

Start with the players to get a feel for Loom, and then start building your own with the modules!

Each module affects the player's behavior in a different way. Some modules determine when a player starts or stops; others specify the rhythm, pitch, and velocity of the generated MIDI notes; yet others make decisions based on the output of other players in the Live set.

  • Trigger begins generating. - If Start on Play is checked, begin generating whenever the Live transport starts
    - By default, Trigger will being playing upon MIDI note input. If Pitch is checked, only the note number specified will be listened to
    - Clicking the Impulse button will also trigger a gesture
  • Phrase decides how many gestures to generate in a single phrase: a random number, or forever.
  • Meter sets the rhythmic meter of a gesture by powers of two.
  • Pitch sets the pitch of all notes in a gesture. - The Gaussian parameter sets the range over three octaves
    - Pitches may be clicked or entered with MIDI In.
  • Count sets the number of notes in a gesture.
  • Follow generates a gesture when the remote player has played a gesture.
  • Echo plays the last gesture from a remote player.
  • Loop repeats the last few gestures in sequence.
* More modules are planned and will be added in future versions of Loom.

Some parameters are reused among different modules:

Each module has a probability that it will be executed upon each gesture generation. When a module is executed, the Module Probability pane flashes.

A Gaussian parameter generates a random number, weighted toward a mean. - The Deviation specifies how far from the mean the random values may land
- The Inertia specifies how far each generated value will be from the last generated value. The center and deviation may be altered by clicking directly in the graph box.

A remote parameter points to another player within the open Live set, with a name such as "Track > Rack".

Players generate gestures, which are, simply put, collections of timed MIDI events, of any duration and containing any number of MIDI notes.

All gestures begin life as a single pulse (a half note at middle C), and take shape as they are processed by modules.

Modules are always executed from left to right, meaning that modules which come later in the chain may process and even overwrite the gesture output of earlier modules.

A given track may contain multiple Loom Players. Multiple Loom Player racks may be nested in a parent rack (as is the case in several example Players).
